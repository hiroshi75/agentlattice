# Create an Asset

Generate custom sales assets tailored to your prospect, audience, and goals. Supports interactive landing pages, presentation decks, executive one-pagers, and workflow/architecture demos.

## Triggers

- "create an asset", "build a demo", "make a landing page", "mock up a workflow"
- When you need a customer-facing deliverable for a sales conversation

## Overview

This skill creates professional sales assets by gathering context about:
- **(a) The Prospect** — company, contacts, conversations, pain points
- **(b) The Audience** — who's viewing, what they care about
- **(c) The Purpose** — goal of the asset, desired next action
- **(d) The Format** — landing page, deck, one-pager, or workflow demo

## Format Options

| Format | Description | Best For |
|--------|-------------|----------|
| **Interactive landing page** | Multi-tab page with demos, metrics, calculators | Exec alignment, intros, value prop |
| **Deck-style** | Linear slides, presentation-ready | Formal meetings, large audiences |
| **One-pager** | Single-scroll executive summary | Leave-behinds, quick summaries |
| **Workflow / Architecture demo** | Interactive diagram with animated flow | Technical deep-dives, POC demos |

## Process

### Phase 0: Context Collection
Gather seller context, prospect details, audience type, purpose, and format preference.

### Phase 1: Research
Research prospect company, leadership, brand colors, industry context, and analyze any uploaded materials.

### Phase 2: Structure
Select section layout based on format and purpose. Adapt for audience type (executive, technical, operations, mixed).

### Phase 3: Content Generation
All content should:
- Reference specific pain points from user input or transcripts
- Use prospect's language and terminology
- Map seller's product to prospect's needs explicitly
- Include proof points where available
- Feel tailored, not templated

### Phase 4: Visual Design
Apply prospect brand colors on dark theme, professional typography, smooth animations, responsive layout.

### Phase 5: Clarifying Questions
Summarize understanding and ask targeted questions before building.

### Phase 6: Build & Deliver
Generate self-contained HTML file with all CSS/JS inline. No external dependencies.

### Phase 7: Iteration
Support rapid adjustments to colors, sections, messaging, format, and detail level.

## Content Principles

- Reference specific pain points from conversations
- Use prospect's exact language where possible
- Map seller's product to prospect's stated needs
- Include proof points (case studies, metrics, quotes)
- Feel tailored, not templated

## Quality Checklist

- Prospect company name spelled correctly throughout
- Leadership names are current
- Pain points accurately reflect input
- Seller's product accurately represented
- No placeholder text remaining
- Brand colors applied correctly
- All interactive elements work
- Tone matches audience
